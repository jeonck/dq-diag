# 데이터베이스 품질 진단 툴 - 프로젝트 요약

## 📋 프로젝트 개요

공공데이터 품질관리 매뉴얼을 기반으로 한 **Streamlit 기반 데이터베이스 품질 진단 도구**입니다.

## 🎯 주요 기능

### 6대 품질 진단 지표

| 지표 | 주요 검사 항목 | 심각도 |
|------|---------------|--------|
| **완전성** | 기본키, NULL값, 미사용컬럼, 데이터타입 | 높음 |
| **일관성** | 컬럼명규칙, 중복데이터, 코드값, 날짜형식 | 중간 |
| **정확성** | 도메인, 범위, 형식, 날짜유효성, 논리관계 | 높음 |
| **보안성** | 개인정보노출, 민감정보, 암호화필요성 | 높음 |
| **적시성** | 데이터최신성, 갱신주기, 미래날짜 | 중간 |
| **유용성** | 데이터충분성, 컬럼유용성, 다양성 | 낮음 |

## 🔍 세부 진단 항목

### 1. 완전성 (Completeness) - 70점 기준
- ✅ 기본키(PK) 미정의 검사
- ✅ NULL 값 비율 분석
- ✅ 미사용/무의미 컬럼 탐지
- ✅ 데이터 타입 불일치 검사

**점수 산출:**
- 완전성 비율 × 70% + (30 - 이슈수×5) × 30%

### 2. 일관성 (Consistency) - 60점 기준
- ✅ 컬럼명 명명 규칙 (공백, 특수문자, 대소문자)
- ✅ 데이터 타입 일관성
- ✅ 중복 레코드 탐지
- ✅ 코드값 대소문자 일관성
- ✅ 날짜 형식 일관성

**점수 산출:**
- (100 - 중복률×2) × 60% + (40 - 이슈수×5) × 40%

### 3. 정확성 (Accuracy) - 70점 기준
- ✅ 도메인 정확성 (Y/N, 코드값)
- ✅ 범위 정확성 (음수, 비율, 나이)
- ✅ 형식 정확성 (이메일, 전화번호)
- ✅ 날짜 유효성
- ✅ 논리적 일관성 (시작일 < 종료일)

**점수 산출:**
- 정확성 비율 × 70% + (30 - 이슈수×4) × 30%

### 4. 보안성 (Security)
- ✅ 개인정보 패턴 탐지 (주민번호, 이메일, 전화번호, 카드번호)
- ✅ 민감정보 탐지 (비밀번호, 계좌, 건강정보)
- ✅ 암호화 필요 데이터 (10자리 이상 숫자)

**점수 산출:**
- 100 - 민감정보수×10 - 이슈수×5

### 5. 적시성 (Timeliness)
- ✅ 데이터 최신성 (마지막 수정일 기준)
- ✅ 갱신 주기 규칙성
- ✅ 미래 날짜 검증

**점수 산출:**
- 기본점수(100) - 경과일수감점 - 이슈수×5

### 6. 유용성 (Usability)
- ✅ 레코드 수 충분성
- ✅ 컬럼별 값 채움율
- ✅ 데이터 다양성 (카디널리티)

**점수 산출:**
- 기본점수(100) - 데이터양감점 - 컬럼유용성감점 - 이슈수×3

## 💡 사용 시나리오

### 시나리오 1: 샘플 데이터로 빠른 테스트
1. 사이드바에서 "📂 샘플 데이터 사용" 클릭
2. 기본 선택된 지표(완전성, 일관성, 정확성) 유지
3. "🚀 진단 시작" 클릭
4. 결과 확인 및 이슈 분석

**예상 결과:**
- 전체 품질 점수: 약 60-70점 (보통 등급)
- 주요 이슈: 15-20개 발견
- 처리 시간: 1-2초

### 시나리오 2: 실제 데이터 진단
1. 자신의 CSV 파일 업로드
2. 필요한 진단 지표 선택
3. 진단 실행
4. JSON 리포트 다운로드

### 시나리오 3: 보안 중심 진단
1. 보안성 지표만 선택
2. 개인정보 포함 데이터 진단
3. 민감정보 노출 확인
4. 암호화 필요 항목 파악

## ⚠️ 제약 사항

### 1. 데이터 크기
- 권장: 100MB 이하의 CSV 파일
- 대용량 파일은 샘플링 권장

### 2. 파일 형식
- CSV (UTF-8 인코딩) 형식만 지원
- 첫 행에 헤더(컬럼명) 필수

### 3. 진단 범위
- 구조적 품질 중심
- 비즈니스 규칙은 샘플 수준
- 실시간 데이터 검증은 미지원

### 4. 언어
- 한글/영문 컬럼명 지원
- 일부 패턴은 한글 키워드 기반
